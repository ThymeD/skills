# summary-qa（总结与要点测验）

## 职责边界

**本 Skill 负责**

- 把输入材料整理成**可复习的学习包**：**单选题 / 判断题**（按知识性质选用，不强制比例）+ 每题**出处** + 每题**质量等级（高 / 中 / 低）**（判定口径与**主题/中心思想契合度、重点知识点**等维度见 [rubric.md](rubric.md)「题目质量等级」）。文章总结按**用户可见流程**在答题之后再输出（见下文「用户可见交互流程」）。
- **默认与用户意图**：无相反说明时，遵循上文「默认行为」。用户在本轮（或同一任务连续上下文）**明确指定**的流程或产出（例如只要总结、一次性题面、同一条消息内合并输出题面与解析）**优先于**默认步骤形态；**含糊表述**（如「快点」）不视为覆盖默认，可简短追问或按默认行为执行。**客观环境约束**（无法取得 URL 正文、不可写 `{baseDir}/data/` 等）优先于用户「必须落盘」类指令，并须说明原因与可行替代。
- **出题数量**：默认 **5～10 道题**（单选 + 判断合计）。若材料过短，或**可独立命题、不重复、不牵强**的知识点不足以达到 5 道，可据实**减少题数**，并在步骤 1 或 Coverage 中**简要说明原因**。若用户**指定题数**且 **N ≤ 20**，优先按用户意见执行（仍受材料可支撑度约束时可建议调整）。**题数门槛（强制）**：用户在本轮提出的**目标题数 N**（含显式数字，或「八十道」「尽量多」等可合理换算、推断出的目标数量）若 **N > 20**，**必须先与用户确认**是否仍按该目标执行；**未经确认不得进入步骤 2**，不得输出任何题目的题干与选项。用户确认后**允许**按约定题数生成。若用户要求的题数**明显超出材料可支撑范围**（即使 N ≤ 20），仍须走下文**「先报价再交货」**，**不得**无说明地硬凑低价值题目。
- **先报价再交货（默认响应策略；强制）**：一旦触发「须先与用户协商题数」（**N > 20**，或 **目标 N 相对材料明显偏高**），代理**不得**默认输出题面；首轮协商回复须**至少**包含：**(1) 建议题数 N'** 及**为何**（结合材料体量、可独立考点）；**(2) 若用户仍坚持原目标 N** 时**可预期后果**（如重复考点、表述变体增多、细枝末节题增多、「低」质量题占比上升等，与 [rubric.md](rubric.md) 一致）；**(3) 请用户明确**是否采用 **N'**、是否仍按 **N**、或其它 **N''**。**收到用户明确答复前不得进入步骤 2**。用户确认或改用 N' / N'' 后，再进入步骤 1 → 步骤 2（「交货」）。
- **题目质量标注（强制）**：无论题数多少，**每一道题**在步骤 2 展示题面时须标注 **质量：高**、**质量：中** 或 **质量：低**（三选一，与 rubric 定义一致）；步骤 3 的逐题明细表中须包含同一题号的**相同**等级，便于用户对「题海」中的信息增量心中有数。
- **题目呈现顺序（默认）**：向用户展示时**默认**按质量 **高 → 中 → 低** 排列题号（**第 1 题**为卷内最重要：最贴近主题、中心思想或**作者最希望读者掌握**的要点；**第 N 题**相对最弱）。命题、排序与例外见 [rubric.md](rubric.md)「题目呈现顺序」「质量判定的主要参考维度」；用户明确要求其它顺序时以用户为准。
- **题面展示节奏（强制）**：**默认**在步骤 2 中**每次对用户可见输出仅包含一道题**的题干与选项；须待用户就该题作答（或明确表示跳过该题）后，再展示下一题。**禁止**因题数较少（例如仅 2～3 道）、或用户只说了「出 N 道题」「一共 N 题」而**未**要求一次性展示，就自行在同一条消息中贴出多道题。**例外**：仅当用户在本轮**主动提出**「一次性全部显示」类意图（措辞见步骤 2「一次性题面」）时，才允许一次性列出全部题面。
- 当输入为 **URL** 时：明确要求运行环境先**获取该 URL 的有效正文**；若无法获取，说明原因并请用户粘贴正文或导出可读文本后再继续。
- 当输入为**本地文件路径**时：与 URL 同级处理——**不由本 Skill 直接读盘**；须先通过**工具或运行环境**读取文件并得到**可读文本**（含必要的格式解析，如 PDF/HTML/Markdown 等），再基于该文本执行总结与出题。若无法读取（无权限、路径不存在、二进制无法解析等），说明原因并请用户改路径、换格式或粘贴正文。
- 当输入为**已给出的正文**（用户粘贴、或上一步已读入的完整文本）时：直接基于该文本执行总结与出题。
- **题后**按需输出「核对与争议点」：仅当存在**非常低级、明显误导**时再简短提醒，并附依据；**无则省略**。该部分内容**不得用于出题**。
- 引导用户完成**勾选式反馈**（理由可选），并基于历史反馈提出**优化建议**；**写入 `{baseDir}/data/memory.md` 前必须经用户明确确认**。

**本 Skill 不负责**

- 不实现网络请求、登录、渲染、本地文件系统读写、PDF/HTML 解析等；这些由运行环境（如 Cursor、OpenCode、OpenClaw 及配套工具）完成。
- 不以「全文均匀覆盖」为目标；以**文章核心观点与复习价值**为导向。超长或上下文不足时，必须给出**覆盖范围声明**。

## 何时加载本 Skill

在以下场景应加载并遵循本 Skill：

- 用户提供**文章链接、本地文件路径、粘贴正文、或文档片段**，并希望获得**中文总结 + 测验题**以巩固要点。
- 用户明确提到「总结」「出题」「复习」「单选」「判断」「要点」等与**提炼+测验**相关的需求。

## 默认行为（无相反说明时照此执行）

1. **流程**：多轮交互；步骤 1→2→3→4 常在同一会话中分多条消息完成。**答题前**不向用户展示步骤 3 的答案表与步骤 4 的完整总结。
2. **题面**：每一则对用户回复中，步骤 2 **只展示一道题**；用户答完再发下一题。除非用户**已主动要求**「一次性题面 / 全部题面 / 统考」等（见步骤 2），否则必须逐题。
3. **落盘**：见下方「数据文件」中的落盘降级说明。

### 用户明确指定时的优先级

- 用户**明确指令**与上列默认推荐或下文步骤流程冲突时，**以用户为准**。
- **含糊表述**不视为覆盖默认；可简短追问或按**默认推荐**执行。
- **客观限制**优先于用户意图：无法读 URL/路径、沙箱禁止写 `data/` 等，须说明原因并给出替代（粘贴正文、对话内反馈等）。

### 常见意图（示例）

- **只要总结、不要测验**：可省略步骤 2～3，直接输出等价于步骤 4 的总结（与上节「用户优先」一致）。
- **一次性题面**（须用户**主动提出**）：步骤 2 可一次性列出全部题面（仍**不**附答案）；用户作答后再步骤 3～4。未主动提出时，仍按**每次一题**。
- **单次回复内推进**（或无逐题条件）：仍须遵守「答题前不输出步骤 3～4」。可压缩为「步骤 1 通知 → 步骤 2 全部题面（**仅当**用户已在本轮要求一次性题面）→ 用户在同一轮或后续消息中提交答案 → 步骤 3～4」。**不得**跳过用户作答而直接输出判分与完整总结（**除非**用户明确放弃测验、仅要总结）。
- **题面与解析同条合并输出**（用户明确要求）：**允许**，以用户为准；可**可选**附一句自测提示（例如先自答再对照），**不**强行覆盖用户意图。

## 语言规范

- **输出语种（强制）**：无论输入材料为何种语言（**含英文或其它外文 URL、页面正文**），**对用户可见的全部内容**（步骤说明、完成通知、题干、选项、判分表、逐题解析、学习总结等）**默认须使用简体中文**。**禁止**因原文为英文或其它语言而**整段改用英文**（或其它非中文）作为面向用户的主输出。若用户在本轮**明确要求**使用英文或其它语言作答或总结，依用户为准。
- **题干与选项**：须用**中文**表述考查点；可基于外文材料**转述**为中文。出处中的**摘录**可保留原文以便核对；必要时在摘录后**括注中文**或另起一句中文说明其含义。
- **Skill 指令与用户可见输出**（说明文字、总结、题干、选项、解析）**默认使用简体中文**。
- **例外**（保留原文或非中文）：代码与命令、专有名词/品牌、API 与字段名、数学与符号、**出处摘录**（保持原文以便核对）、法规编号、用户明确要求使用的语言。
- 专有名词可**首次出现中英并列**，后续以中文叙述为主。

细则见 [rubric.md](rubric.md)。

## 内部准备（对用户不可见或不在首轮展示）

代理可先在内部完成：归纳要点、命题、记录每题正确答案与解析、Coverage 等；但**对用户的可见输出**必须严格遵循下方「用户可见交互流程」，**禁止**在答题结束前一次性贴出「带答案的题卷」或全文总结。

- **出题数量**：须符合上文「职责边界」中的规则；步骤 1 中的 **N** 与之一致。**步骤 0** 触发时须已**拍板**（含 N>20 确认或 N'/N''），否则不得生成题面。
- **题目质量等级**：命题时为每题判定 **高 / 中 / 低**，与对外展示一致；档位依据见 [rubric.md](rubric.md)「质量判定的主要参考维度」。
- **题目排序与题号**：在判定 **高/中/低** 之后，**先**按 **高 → 中 → 低** 排定全卷顺序，**再**赋予 **第 1～N 题** 题号（与步骤 2 展示顺序一致）。细则见 [rubric.md](rubric.md)「题目呈现顺序」。
- **选项顺序与正确答案字母**：每道题在确定题干与选项**内容**后，须将选项**打乱**再标注 A/B/C/D，使正确答案**不要系统性落在同一字母**（常见错误是「正确答案全是 B」）；判断题中「对/错」与 **A/B** 的对应关系也须**随机化**。细则见 [rubric.md](rubric.md)「正确答案的位置」。

## 用户可见交互流程（必须按顺序；核心：先考后讲、先判后总结）

下列步骤为**面向用户的输出顺序**。前序步骤未完成时，**不得**提前输出后续步骤中的内容（用户明确要求跳过测验时除外，须在回复中说明已跳过）。

### 步骤 0（须协商题数且未拍板时；优先于步骤 1）

触发条件（满足**任一**且尚未获得用户**明确拍板**）：**(A)** 目标题数 **N > 20**（含合理推断）；**(B)** 目标 **N 相对材料明显偏高**（即使 N ≤ 20）。「明确拍板」指用户明确采用 **N'**、坚持 **N**、或给出其它 **N''**（例如「就按 12 道出」）。

- 本步对应职责边界中的**先报价再交货**：对用户**仅**允许输出**报价**内容——**(1) 建议题数 N'** 及理由；**(2) 若坚持原 N 的后果**（重复考点、表述变体、细枝末节题增多等，与 rubric 中质量等级说明一致）；**(3) 请用户确认**采用 N'、坚持 N 或给出 N''。**严禁**输出任何题目正文、选项、正确答案、解析、完整文章总结、或「核对与争议点」。
- 若仅因 **(A) N > 20** 而进入本步，用户**改为 N' ≤ 20** 即视为已拍板，可进入**步骤 1**；若用户**明确坚持 N > 20**，须得到**一句明确确认**（如「确认按 80 题出」）后方可进入**步骤 1**（步骤 1 中的 N 以拍板为准）。
- 用户完成拍板后，再进入**步骤 1** 与后续流程。

### 步骤 1：完成通知（首轮只允许这些内容）

- 用简短说明告知：已理解材料并完成**出题**；**共 N 道题**（如有需要可注明单选 / 判断各多少）。**可**注明题目**按重要性从高到低**排列（**第 1 题**优先对应主旨与重点知识点，见 [rubric.md](rubric.md)）。若 N 因材料短或知识点不足而少于 5，或已与用户确认后调整题数，可在此**一句说明**。若采用非默认题序（用户要求），须**一句说明**。
- 若存在 **Coverage（覆盖声明）**，用一两句话说明即可。
- **严禁在本步输出**：任何题目正文、选项、正确答案、解析、完整文章总结、或「核对与争议点」。

### 步骤 2：作答（默认逐题；一次性题面须用户事先声明）

**默认模式：逐题（强制；不要二选一）**

- **步骤 1 结束后，直接进入逐题流程**：下一则对用户输出应**仅**包含**第 1 题**的题干与选项（或先一句「下面从第 1 题开始」再贴题面），**禁止**要求用户在「逐题 / 一次性题面」之间做选择或回复「开始」才能看题（用户可直接用选项字母/数字作答）。
- **题号与质量顺序**：**第 1～N 题**已按质量 **高 → 中 → 低** 排定（**第 1 题**为卷内最重要，最贴近主题、中心思想或重点知识点；见 [rubric.md](rubric.md)）。逐题展示时按 **第 1 题 → 第 2 题 → …** 推进；**每一则用户可见消息内**只展示**一题**的**题干 + 选项**，用户答完再发下一题。**一次性题面**时，题目列表顺序须同为 **高→中→低**。
- **质量标注**：每一题须带**质量：高**、**质量：中** 或 **质量：低**（与 [rubric.md](rubric.md) 定义一致）。
- **题面排版（推荐；步骤 2 对用户展示）**：采用统一「标题行 + 题干 + 选项」结构，便于扫读与 CLI/TUI 选择。完整形态见 [examples.md](examples.md)「推荐题面排版」。
  - **第 1 行（题面标题行）**：`第 N 题（质量：高｜中｜低）｜单选题` 或 `第 N 题（质量：高｜中｜低）｜判断题`。分隔符 **｜**（U+FF5C 全角竖线）用于隔开题型；**勿**在标题行写题干正文。
  - **空一行**。
  - **题干**：单独成段，可含引号与标点；**勿**与标题行挤在同一行。
  - **选项**：**单选题**为 **A. B. C. D.** 四项，**每项独占一行**，选项字母与正文之间用句点 **.** 或 **．** 均可；**判断题**仍为 **A、B** 两项（文案如「正确 / 错误」），同样每项一行，并与上文「选项标签」一致。
  - **一次性题面**时，每一道题均重复上述结构（标题行 → 空行 → 题干 → 选项），题与题之间可用空行分隔。

**一次性题面（例外；须用户主动提出）**

- **仅当用户在本轮调用 skill 的提示里已明确要求一次性展示全部题目**时，才在步骤 2 **一次性**列出全部题目的题干与选项（仍**不得**附答案与解析）。可接受的表述示例包括但不限于：「一次性列出全部题目」「全部题面再答」「一次性全部显示」「统考模式」「我要一张卷子」「同一屏出完所有题」等；**同义明确意图**亦可，**含糊表述**（如「快点」「题少就一起出」）**不**视为授权一次性题面，可简短追问或仍按逐题执行。
- **以下不视为**一次性授权，代理**仍须逐题**：仅指定题数（如「出 5 道题」「来 2 题」）、题数 ≤ 5、或任何**未**表达「希望本条/本轮一次性看到全部题干」的说法。
- 用户**未**作上述声明时，**不得**主动提供一次性题面，也**不得**用问卷形式让用户先选答题方式。

**选项标签（展示）**

- **单选题**：固定使用 **A、B、C、D** 四项，后接选项正文；与「题面排版（推荐）」一致时，采用 **`A.` … `D.`** 每项一行。
- **判断题**：同样固定使用 **A、B** 两项（例如 A=正确、B=错误，或 A=是、B=否，须与题干表述一致），**不要**只写「正确 / 错误」而不带 A/B 标签，以便与单选题交互方式一致。
- 题目应便于 CLI/TUI **鼠标或键盘**选择；选项字母与数字对应关系见下「用户输入如何解析」。

**其他**

- **严禁在本步输出**：正确答案、「本题选…」、解析、文章总结；也不得在展示第 k 题时泄露其他题的答案。
- 若用户在**步骤 2 中途**临时要求改为一次性贴剩余题面，可按其要求调整（仍不得附答案）。

**用户输入如何解析（代理判题前必须统一）**

- **字母选项**：用户输入 **A、B、C、D**（含 **a、b、c、d**）均视为对应选项，**大小写不敏感**。
- **数字选项**：用户输入 **1、2、3、4** 分别对应 **A、B、C、D**（仅适用于**单选题**的四个选项）。
- **判断题（仅 A/B）**：用户输入 **1、2** 分别对应 **A、B**（与上条数字规则一致，避免两套逻辑）。
- **歧义或无法认定时**：若用户一次回复中同时出现多题答案、含糊词、无法映射到唯一选项，或与当前题号不匹配，须**先与用户二次确认**（例如：「请确认本题是否选 B（第二项）？」），待用户明确后再记录本题并进入下一题；不得猜测。

### 步骤 3：判分与讲评（用户全部作答完毕后）

本步输出分**两段**：先**表格**汇总结果，再**按题号**给出解析（列表形式，与表格不重复罗列题干）。

**（1）总体结果（一张表）**

| 总题数 | 答对 | 答错 | 得分率 |
|:------:|:----:|:----:|:------:|
| N | M | N−M | 百分比或分数 |

**（2）逐题明细（一张表）**

列固定为：**题号** | **题型** | **质量** | **结果** | **完整题干** | **你的选择** | **正确答案**。（**质量**列为 **高 / 中 / 低**，须与步骤 2 该题标注一致。）

- **结果**：✓ / ✗，或「对」「错」，全文统一一种即可。
- **完整题干**：与步骤 2 中向用户展示的题干**一致**（含标点），不得改用摘要。
- **你的选择** / **正确答案**：格式为 **`字母 — 该选项的完整文字`**（例如 `C — 主动失效（……）`）；判断题同样写出 **A/B 对应的完整文案**（如 `A — 错误`）。
- **长文本表格**：题干或选项文字较长时，优先使用 **HTML `<table>`**，在单元格内用 `<br>` 换行；若环境不支持 HTML，可用 Markdown 管道表并在单元格内使用 `<br>`（以当前渲染器为准）；**不得**因篇幅省略题干或选项全文。

**（3）逐题解析（在两张表之后）**

- 按 **第 1 题、第 2 题…** 分段，每题给出**简短解析**（复习导向，不钻牛角尖）；需要时可附**出处**（章节标题 + 摘录）。
- 解析段**不再重复整张题干**，可用「第 k 题」指代；若解析需引用原文，可摘录短语。

- **严禁在本步之前**输出上述表格、正确答案与解析。

### 步骤 4：文章总结

- 在步骤 3 结束后再输出**完整的学习总结**（核心观点、结构化要点等），作为对材料的**归纳**，供用户对照复习。

### 步骤 5：文章明显问题提醒（可选）

- **有则提醒，无则省略**；仅在存在**非常低级、明显误导**时，用简短文字提醒，并附「章节标题 + 摘录」；不过度较真。
- 该内容**不得用于出题**；且放在**步骤 4 之后**，避免干扰答题与判分。

### 步骤 6（可选）：反馈与记忆

- **反馈收集**：按 [schemas.md](schemas.md) 中的 `FeedbackItem` 引导勾选；理由可选。
- **优化建议**：可基于 `{baseDir}/data/feedback.jsonl` 与本轮反馈列出建议；**仅当用户确认**后，将约定写入 `{baseDir}/data/memory.md`。

## 数据文件（均在技能目录下）

`{baseDir}` 指 SKILL.md 所在目录（技能根目录）。

| 路径 | 用途 |
|------|------|
| `{baseDir}/data/memory.md` | 用户确认后的长期偏好与约束 |
| `{baseDir}/data/feedback.jsonl` | 每行一条 JSON 反馈记录，追加写入 |
| `{baseDir}/data/cases/` | 高质量题目案例（晋升规则见下） |
| `{baseDir}/data/archive/` | 退役案例归档 |

**案例晋升（简版）**：用户多次正向反馈或明确标记为范例的题目，可摘要写入 `{baseDir}/data/cases/`；过时或多次负向反馈的可移至 `{baseDir}/data/archive/`。

**落盘降级**：若无写权限或目录不存在且无法创建，则跳过落盘，仅在对话中收集反馈或偏好，并一句说明未写入本地文件。

## 与环境的协作（URL 与本地文件）

两类输入在调用本 Skill **之前**，都应先变成「一段可用的正文」，流程与责任划分一致。

**URL**

1. 先通过环境能力获取**可读正文**。
2. 若失败：明确告知，并请用户**粘贴正文**或使用其他导出方式。
3. 获取成功后：再加载本 Skill 并执行总结与出题。

**本地文件路径**

1. 先通过**读文件 / 打开文档**等工具读取路径对应内容，并转为**可读纯文本**（必要时走环境支持的 PDF、HTML、Office、Markdown 等解析）。
2. 若失败：明确告知（权限、路径、编码、格式不支持等），并请用户修正路径、换格式或**粘贴正文**。
3. 读取成功后：将**正文**作为输入再加载本 Skill 并执行总结与出题（可在 `source_ref` 中记录原始路径，见 [schemas.md](schemas.md)）。

**已具备正文时**（含用户粘贴、或上一步已读入的文本）：直接进入总结与出题，无需再读 URL 或路径。
