# 示例（summary-qa）

以下为**虚构短文**的演示，展示字段形态；真实使用时应替换为用户的材料与真实出处摘录。

**对用户输出时**：勿在首轮把「题目 + 正确答案」一并贴出；应按 SKILL 顺序：先通知题量 → 再逐题仅题面 → 用户答完后再展示 `correct` 与 `explanation`。

**本地文件路径输入时**：代理应先用工具读入正文，再在 `StudyPack` 中标注来源，例如：

```yaml
source_type: file
source_ref: "D:/project/docs/architecture.md"   # 或工作区相对路径；以实际读到的路径为准
```

正文内容来自该次读取结果；若路径仅作入口、实际还合并了多文件，可在 `source_ref` 中简要说明。

## 推荐题面排版（步骤 2，对用户）

下列为**推荐**展示形态（与 [SKILL.md](SKILL.md) 步骤 2「题面排版（推荐）」一致）；代理命题后仍须打乱选项字母与正确答案位置，下例仅为版式参考。


**判断题示例（标题行将「单选题」换成「判断题」；选项仍为 A / B）**


## 示例 1：单选题

**材料片段（假设）**  
小节标题：《缓存失效的三种策略》  
正文：「主动失效适用于变更可预期的场景；TTL 适用于可容忍短期不一致的业务……」

**题目**（选项顺序已打乱，正确答案为 **C**，避免示例也全是 B）

- `id`: `q-demo-1`
- `type`: `single`
- `stem`: 在变更可预期、需要尽快保持一致时，文中更侧重采用哪种方式？
- `options`:  
  - A：仅依赖 TTL 到期  
  - B：永远不做失效  
  - C：主动失效  
  - D：随机失效  
- `correct`: `C`
- `explanation`: 文中将「主动失效」与「变更可预期」对应；应选「主动失效」对应项（本题标为 C）。
- `source_ref`:  
  - `section_title`: 《缓存失效的三种策略》  
  - `excerpt`: 「主动失效适用于变更可预期的场景；TTL 适用于可容忍短期不一致的业务……」

## 示例 2：判断题（展示为 A / B）

**对用户展示（步骤 2）**（本示例将「错误」放在 **A**、「正确」放在 **B**，避免与示例 1 的正确字母雷同）

- `stem`: 根据上文，TTL 策略适用于「绝对不能容忍任何不一致」的金融实时对账场景。（判断正误）
- `options`: **A** 错误　**B** 正确

**对内字段**

- `id`: `q-demo-2`
- `type`: `boolean`
- `correct`: `A`（陈述为假，应选「错误」；用户答 `a` / `1` 均视为选 A）
- `explanation`: 文中写明 TTL 适用于可容忍短期不一致；与「绝对不能容忍不一致」矛盾。
- `source_ref`: 同上摘录。
