# 技能审查器

分析已安装技能的质量问题和优化机会。

## 技能分类

### 远程技能 (~/.agents/skills/)
- 通过 `npx skills add` 安装
- 来源：GitHub 仓库
- **行为**：仅审查，不修改

### 本地技能 (~/.config/opencode/skills/)
- 手动创建或 fork
- **行为**：完整审查 + 优化建议 + 确认后执行修改

## 触发条件

在以下情况使用：
- 安装新技能时验证其质量
- 检查现有技能的改进点
- 审查技能文档完整性
- 检查安全问题或性能问题

## 使用方法

```
# 审查指定技能
"审查技能 [skill-name]"
"检查 [skill-name] 有什么问题"

# 审查所有技能
"审查所有已安装技能"
"检查我的技能有什么问题"
```

## 分析清单

### 1. 文件结构检查

验证技能是否包含必要文件：
- `SKILL.md` - 技能主定义（必需）
- `AGENTS.md` - 智能体详细指令
- `README.md` - 文档（推荐）
- `rules/` - 规则文件（如适用）

### 2. SKILL.md 质量检查

检查项：
- Front matter 包含 `name`, `description`, `license`, `metadata`
- 清晰的 `## 触发条件` 部分
- 详细的使用说明
- 必要的示例

### 3. AGENTS.md 质量检查

检查项：
- 全面的文档
- 带解释的代码示例
- 清晰的部分和组织
- 版本信息和归属

### 4. 安全检查

标记潜在问题：
- 代码中的外部 API 密钥或密钥
- 未清理的用户输入
- 缺少输入验证
- 危险的系统命令

### 5. 文档质量评估

评估：
- 目的是否清晰？
- 是否提供了示例？
- 使用上下文是否解释？
- 限制条件是否记录？

### 6. 代码质量（如适用）

检查项：
- 适当的错误处理
- 类型安全（TypeScript）
- 性能考虑
- 浏览器兼容性

## 输出格式

### 远程技能（仅审查）

```markdown
## 技能审查: [skill-name] 🔒

**路径:** `~/.agents/skills/[skill-name]/`
**类型:** 远程（只读）

### 结构评估
- ✅ 有 SKILL.md
- ⚠️ 缺少 AGENTS.md

### 质量评分: 7/10

### 发现的问题
1. [问题描述] - [严重性: 高/中/低]

### 建议
- [建议内容]
- （远程技能不会自动修改）
```

### 本地技能（审查+优化）

```markdown
## 技能审查: [skill-name] ✏️

**路径:** `~/.config/opencode/skills/[skill-name]/`
**类型:** 本地（可优化）

### 结构评估
- ✅ 有 SKILL.md
- ✅ 有 AGENTS.md

### 质量评分: 8/10

### 发现的问题
1. [问题描述] - [严重性: 高/中/低]

### 优化建议
| # | 问题 | 修复 | 应用? |
|---|------|------|--------|
| 1 | 缺少 README | 创建 README.md | [ ] |

### 下一步
说"应用修复 #1"或"全部应用"来执行修改。
```

## 严重性等级

- **高**：安全风险、功能缺失、关键文档缺失
- **中**：模式不佳、文档不完整
- **低**：样式改进、细微遗漏
