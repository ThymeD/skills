# Life Audit 人生自检技能

基于 Dan Koe 《如何在一天内改变你的整个人生》，帮助用户进行深度自我审视。
支持分阶段推进，本地持久记录每次回答，分析用户当前处境并给出针对性建议。

## 初始化

**首次使用时执行：**

1. 检查本地是否已有记录文件（`~/.life-audit/records.json`）
   ```bash
   ls ~/.life-audit/records.json 2>/dev/null && echo "EXISTS" || echo "NEW"
   ```

2. 若无记录，创建目录和初始文件：
   ```bash
   mkdir -p ~/.life-audit
   ```
   然后创建 `~/.life-audit/records.json`，初始内容：
   ```json
   {
     "version": "1.0",
     "sessions": []
   }
   ```

3. 向用户说明三阶段结构，询问今天想做哪个阶段。

## 读取记录

每次运行时，先读取现有记录：

```bash
cat ~/.life-audit/records.json
```

根据记录内容判断：
- **无任何session** → 引导进入第一阶段第一组
- **有未完成的session** → 询问是继续上次，还是开始新一轮
- **有完整记录** → 展示上次完成时间和阶段，询问用户意图

展示历史摘要时，格式如下：
```
📋 你有 N 次自检记录
最近一次：YYYY-MM-DD  阶段：[阶段名]  完成度：X/22
```

## 三阶段结构

### 第一阶段：探索痛苦与反愿景（问题 1-11）

**适合场景**：第一次做，或重新开始一轮。建议在安静的下午，纸笔并用。

分两组推进，不要一次性抛出所有问题：

**第一组（当下的痛苦，问题1-4）**
逐题引导，等用户回答后再给下一题。每题回答后，给一句简短的反射性回应（不评判，不给建议），帮用户感觉被听见。

**第二组（反愿景，问题5-11）**
告知用户："这一组会让你感到不舒服，这是正常的，也是必要的。"
同样逐题推进。

完成第一阶段后，**不要立即进入第二阶段**。告知用户：
> "建议你把这些答案放一放，至少睡一觉。一周后再做第二阶段效果更好。"

### 第二阶段：最小可行愿景（问题13-15）

**进入条件**：距离第一阶段完成至少3天（检查记录中的时间戳）。
若间隔不足，温和提示，但尊重用户选择继续。

先让用户回顾第一阶段的答案（从记录中提取摘要展示）。
然后引导问题13-15。

### 第三阶段：全天打断 + 整合（问题16-22 + 定时提醒设置）

**每日打断问题**（问题 D1-D6）：
询问用户是否要设置提醒。如果是 macOS/Linux，提供 cron 设置命令（见下方）。

**整合问题**（问题16-22）：
建议每周做一次，不必每天。引导用户完成后，执行处境分析。

## 保存记录

每次用户完成一题后，立即保存。格式：

```json
{
  "version": "1.0",
  "sessions": [
    {
      "session_id": "s001",
      "started_at": "2026-04-11T09:00:00",
      "phase": 1,
      "answers": [
        {
          "question_id": "q01",
          "question_text": "那种低沉而持续的不满是什么？",
          "answer": "用户的原始回答",
          "answered_at": "2026-04-11T09:12:00"
        }
      ],
      "analysis": null,
      "completed_at": null
    }
  ]
}
```

保存命令（使用 Python 进行 JSON 操作，避免手动字符串拼接）：

```bash
python3 -c "
import json, sys
with open('$HOME/.life-audit/records.json', 'r') as f:
    data = json.load(f)
# 修改 data
with open('$HOME/.life-audit/records.json', 'w') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
"
```

## 分析建议

完成第二阶段或第三阶段后，读取全部记录，执行处境分析。
详细判断逻辑见 `references/analysis.md`。

分析结果结构：
1. **当前所处阶段**：失调期 / 不确定期 / 发现期
2. **核心模式识别**：从答案中提炼用户反复出现的主题词
3. **最大阻力来源**：基于问题9-11的回答判断
4. **一句话处境描述**：用用户自己的语言回镜
5. **下一步建议**：具体、可执行的1-2件事

分析完成后，将结果写入当前 session 的 `analysis` 字段保存。

## 定时提醒设置（可选）

如果用户在 macOS 或 Linux 上，可以设置 cron 提醒。
告知用户以下命令会修改其 crontab，征得同意后再执行：

```bash
(crontab -l 2>/dev/null; echo "0 11 * * * osascript -e 'display notification \"我现在在做的事情在回避什么？\" with title \"人生自检\"'") | crontab -
```

Windows 用户建议手动设置系统提醒，或使用手机日历。

## 对话风格原则

- **逐题推进**，不要一次抛出所有问题
- **不急于给出建议**，先让用户把话说完
- **不评判答案**，每个回答后给一句简短的反射（"听起来这件事压着你有段时间了"）
- **保持温和但直接**，必要时点出用户在回避什么
- **尊重节奏**，用户说"今天到这里"就停，不催促
- 所有问题中文提问，记录也以中文存储

---

## 错误处理

- 若 `~/.life-audit/` 目录无写权限，提示用户手动创建或更改路径
- 若记录文件损坏（JSON 解析失败），备份原文件并重新初始化
- 若用户跳过某题，记录 `"answer": null`，不强制要求回答
