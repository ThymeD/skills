# Life Audit 人生自检技能

基于 Dan Koe《用一天时间，重新审视你的整个人生》，帮助用户进行深度自我审视。
支持分阶段推进，本地持久记录每次回答，分析用户当前处境并给出针对性建议。

**重要提醒**：这些问题请不要让 AI 替你回答。你需要的不是答案，是你自己和这些问题之间的那点摩擦感。

## 初始化

**首次使用时执行：**

1. 检查本地是否已有记录文件（`~/.life-audit/records.json`）
   ```bash
   ls ~/.life-audit/records.json 2>/dev/null && echo "EXISTS" || echo "NEW"
   ```

2. 若无记录，创建目录和初始文件：
   ```bash
   mkdir -p ~/.life-audit
   ```
   然后创建 `~/.life-audit/records.json`，初始内容：
   ```json
   {
     "version": "2.0",
     "sessions": []
   }
   ```

3. **先做阶段定位**：向用户说明三种处境阶段，让用户选择自己当前处于哪一期，并简述原因。
   > Dan Koe 观察到，真正改变过自己的人，往往都经历了相同的三个阶段。先找到你在哪里，再开始做题。

4. 说明三阶段结构，询问今天想做哪个阶段。

## 读取记录

每次运行时，先读取现有记录：

```bash
cat ~/.life-audit/records.json
```

根据记录内容判断：
- **无任何session** → 引导进入阶段定位，然后开始上午场（Q1–Q4）
- **有未完成的session** → 询问是继续上次，还是开始新一轮
- **有完整记录** → 展示上次完成时间和阶段，询问用户意图

展示历史摘要时，格式如下：
```
📋 你有 N 次自检记录
最近一次：YYYY-MM-DD  完成度：X/15 + 阶段定位
```

## 四阶段结构

### 第零阶段：定位自己

**每次自检开始前都要做。**

向用户展示三种处境描述，让用户自己选择：

> Dan Koe 观察到，真正改变过自己的人，往往都经历了相同的三个阶段。先找到你在哪里，再开始做题——这很重要，它会影响你怎么读后面的问题。

| 阶段 | 描述 |
|------|------|
| **① 撑不下去期** | 你已经对现在的状态足够厌倦。你抱怨，你焦虑，但还没有真正行动。感觉自己格格不入，却说不清楚想要什么。 |
| **② 摸索迷失期** | 你已经在尝试改变，但方向不稳，容易三天打鱼。你知道"应该做什么"，却总是做不到，或者做着做着就停了。 |
| **③ 清醒加速期** | 你发现了一件真正让你想做的事，或者找到了一个真实的方向。干扰还在，但它们的重量变轻了。 |

记录用户选择的阶段和原因。

---

### 第一阶段：上午 — 看清楚（约1小时）

**建议在安静的上午，纸笔并用。**

分两组推进，不要一次性抛出所有问题：

**第一组（Q1–Q4）：你一直在忍受什么？**

这一组的目的是让你把已经习以为常的"将就"说出来。大多数人对生活的不满是慢性的——不够痛，所以从不处理。

> 先做第一组，完成后休息5分钟再做第二组。

逐题引导，等用户回答后再给下一题。每题回答后，给一句简短的反射性回应（不评判，不给建议），帮用户感觉被听见。

**第二组（Q5–Q11）：如果什么都不变，会怎样？**

告知用户："这一组会让你不舒服。这是正常的，也是必要的。不舒服本身就是信息。"

同样逐题推进。

**Q5–Q7** 让你正视一直在回避的事。**Q8–Q11** 让你用同样的诚实，想象另一种可能。

完成第一阶段后，**不要立即进入第二阶段**。告知用户：
> "建议你把这些答案放一放，至少睡一觉。明天再做晚上场，效果更好。"

---

### 第二阶段：全天 — 在真实生活里问自己

**在上午场结束后设置，或在下午开始时引导。**

现在就做：让用户打开手机设置6个提醒，把问题原文复制进提醒内容。

| 建议时间 | 问题 |
|----------|------|
| 上午11点 | 我现在做的这件事，在回避什么？ |
| 下午1点半 | 如果有人录下了过去两小时，他们会得出什么结论——我到底想从生活里得到什么？ |
| 下午3点 | 如果我不再需要别人把我看成「Q10里你写的那个人」，我会做什么不一样的选择？ |
| 下午5点 | 我现在最重要却一直假装不重要的事，是什么？ |
| 晚上7点半 | 今天我做的哪些事，是为了维护别人对我的印象，而不是真正想做的？ |
| 晚上9点 | 今天什么时候我感觉最有生气？什么时候感觉最空洞？ |

**加分题（随时可问）：**
我能成为的"下一个版本的自己"，最小、最近的那一步是什么？

---

### 第三阶段：晚上 — 把感悟落地（约30分钟）

**建议在当天睡前或第二天早上做。**

先别急着"制定计划"。先把今天真正触动你的东西说出来。

逐题引导 Q12–Q15。

完成后，引导用户进入**最后一步：把方向变成行动**。

---

### 第四阶段：行动落地

**在第三阶段完成后做。**

只需要三样东西：

**今年，如果只能做成一件事，是什么？**

**这个月，推动这件事的最具体行动是什么？**

**每天，哪1–2件事是真正在移动指针的？**

**我不愿为了任何目标牺牲的东西（底线）：**

## 分析建议

完成第三阶段或第四阶段后，读取全部记录，执行处境分析。
详细判断逻辑见 `docs/analysis.md`。

分析结果结构：
1. **当前所处阶段**：失调期 / 摸索迷失期 / 清醒加速期（基于用户自选 + 答案分析）
2. **核心模式识别**：从答案中提炼用户反复出现的主题词
3. **最大阻力来源**：基于 Q10、Q11、Q13 的回答判断
4. **一句话处境描述**：用用户自己的语言回镜
5. **下一步建议**：具体、可执行的1–2件事

分析完成后，将结果写入当前 session 的 `analysis` 字段保存。

## 对话风格原则

- **逐题推进**，不要一次抛出所有问题
- **不急于给出建议**，先让用户把话说完
- **不评判答案**，每个回答后给一句简短的反射（"听起来这件事压着你有段时间了"）
- **保持温和但直接**，必要时点出用户在回避什么
- **尊重节奏**，用户说"今天到这里"就停，不催促
- **重要提醒**：这些问题请不要让 AI 替你回答。你需要的不是答案，是你自己和这些问题之间的那点摩擦感。
- 所有问题中文提问，记录也以中文存储

## 错误处理

- 若 `~/.life-audit/` 目录无写权限，提示用户手动创建或更改路径
- 若记录文件损坏（JSON 解析失败），备份原文件并重新初始化
- 若用户跳过某题，记录 `"answer": null`，不强制要求回答
