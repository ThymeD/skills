# Project Init - 智能体详细指令

**版本 1.0.0**

> **问题反馈与贡献**: 使用本技能遇到问题时，请先查阅 [GitHub Issues](https://github.com/ThymeD/skills/issues) 和项目级 [AGENTS.md](../AGENTS.md#问题反馈与贡献指南)。AI 工具应具备自主排查和贡献解决方案的能力。

本文档为智能体提供项目初始化的详细指令。

---

## 概述

Project Init 技能用于创建项目工作目录结构，为项目开发提供必要的目录和文件。

---

## 触发条件

当用户说以下话术时使用此技能：
- 开始开发
- 初始化项目
- 项目初始化
- 创建项目目录
- 开始做项目

---

## 核心职责

### 1. 检测项目状态

检测项目是否已有工作目录：
- 检查 `.poc/` 目录是否存在
- 检查 `requirements/` 目录是否存在
- 检查 `docs/` 目录是否存在
- 检查 `tests/` 目录是否存在

### 2. 创建目录结构

根据需要创建以下目录：
```
project/
├── .poc/                  # POC 验证目录（初始为空）
├── requirements/          # 需求清单
│   ├── SPEC.md           # 需求总览
│   └── features/         # 功能需求
├── docs/                 # 项目文档
└── tests/                # 测试文件
```

注意：如果项目已有 src/ 目录，不要创建；只创建 OpenCode 工作目录。

### 3. 引导用户开始开发

初始化完成后，输出：
1. 创建的目录结构
2. 后续使用说明

---

## 交互流程

```
用户：开始开发 / 初始化项目
    ↓
project-init 检测项目状态
    ↓
判断：工作目录是否已存在
    ↓
已存在 → 提示跳过
    ↓
不存在 → 创建目录结构
    ↓
输出完成信息
    ↓
引导用户开始开发
```

---

## 输出要求

初始化完成后，需报告：
1. 创建的目录结构
2. 当前项目状态
3. 后续使用说明

---

## 重要说明

- 只在项目首次开发时触发一次
- 不会覆盖已有目录
- 初始化是幂等的，多次执行不会有问题
- 如果目录已存在，跳过创建
